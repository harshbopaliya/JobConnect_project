# JobConnect Project

## Overview

The **JobConnect** is a job portal application built using **Django** for the backend and **React** for the frontend. It allows users to apply for jobs, view job listings, and companies to manage their job postings, reviews, and benefits.

## üöÄ Quick Start

### 1. Clone the Repository

```bash
git clone https://github.com/harshbopaliya/JobConnect_project.git
cd JobConnect_project
```

### 2. Set Up the Frontend

> The **frontend** resides in the main project folder.

```bash
# Install frontend dependencies
npm install
# Start the development server
npm run dev
```

Visit the frontend at: [http://localhost:5173](http://localhost:5173)

### 3. Set Up the Backend (Django)

#### Prerequisites

- Python 3.8 or higher
- MySQL
- Virtual Environment

#### Steps

```bash
# Navigate to the backend folder
cd backend
# Create a virtual environment
python -m venv venv
# Activate the virtual environment
# For Windows:
venv\Scripts\activate
# For macOS/Linux:
source venv/bin/activate
# Install required dependencies
pip install django djangorestframework django-cors-headers mysqlclient python-dotenv pillow
# Make migrations and apply them
python manage.py makemigrations
python manage.py migrate
# Create a superuser for accessing the admin panel
python manage.py createsuperuser
# Start the Django development server
python manage.py runserver
```

Visit the Django admin at: [http://localhost:8000/admin](http://localhost:8000/admin)

## Database Structure

The database for the **JobConnect** application consists of several tables, each related to different parts of the system. Below are the tables and their columns, including whether they are auto-generated by Django or created manually.

### Tables and Columns:

#### `accounts_employerprofile`

- `id`: `int` (Primary Key, Auto-increment)
- `position`: `varchar` (Position of the employer)
- `department`: `varchar` (Department of the employer)
- `created_at`: `datetime` (Date and time when the record was created)
- `updated_at`: `datetime` (Date and time when the record was last updated)
- `company_id`: `int` (Foreign key to the company's ID)
- `user_id`: `int` (Foreign key to the user profile)

#### `accounts_jobseekerprofile`

- `id`: `int` (Primary Key, Auto-increment)
- `resume`: `varchar` (Resume link or path)
- `skills`: `longtext` (List of jobseeker's skills)
- `experience_years`: `int` (Number of years of experience)
- `education`: `longtext` (Educational background)
- `bio`: `longtext` (Biography of the jobseeker)
- `location`: `varchar` (Jobseeker's location)
- `created_at`: `datetime` (Creation timestamp)
- `updated_at`: `datetime` (Last updated timestamp)
- `user_id`: `int` (Foreign key to the user profile)

#### `accounts_user`

- `id`: `int` (Primary Key, Auto-increment)
- `password`: `varchar` (Hashed user password)
- `last_login`: `datetime` (Last login timestamp)
- `is_superuser`: `tinyint` (Is the user a superuser?)
- `username`: `varchar` (Username for the user)
- `first_name`: `varchar` (User's first name)
- `last_name`: `varchar` (User's last name)
- `is_staff`: `tinyint` (Is the user a staff member?)
- `is_active`: `tinyint` (Is the user active?)
- `date_joined`: `datetime` (Date the user joined)
- `email`: `varchar` (User's email address)
- `user_type`: `varchar` (Type of user: Employer, Jobseeker, etc.)
- `profile_picture`: `varchar` (Profile picture URL or path)
- `phone_number`: `varchar` (Phone number)

#### `accounts_user_groups`

- `id`: `int` (Primary Key, Auto-increment)
- `user_id`: `int` (Foreign key to user)
- `group_id`: `int` (Foreign key to group)

#### `auth_group`

- `id`: `int` (Primary Key, Auto-increment)
- `name`: `varchar` (Name of the group)

#### `auth_permission`

- `id`: `int` (Primary Key, Auto-increment)
- `name`: `varchar` (Permission name)
- `content_type_id`: `int` (Foreign key to content type)
- `codename`: `varchar` (Permission code name)

#### `companies_benefit`

- `id`: `int` (Primary Key, Auto-increment)
- `company_id`: `int` (Foreign key to company)
- `text`: `varchar` (Description of the company benefit)

#### `companies_company`

- `id`: `int` (Primary Key, Auto-increment)
- `name`: `varchar` (Company name)
- `logo`: `varchar` (Logo URL or path)
- `cover_image`: `varchar` (Cover image URL or path)
- `website`: `varchar` (Website URL)
- `description`: `text` (Company description)
- `mission`: `text` (Company mission)
- `employee_count`: `varchar` (Employee count, e.g., "1-10")
- `location`: `varchar` (Company location)
- `founded_year`: `int` (Year the company was founded)
- `linkedin`: `varchar` (LinkedIn URL)
- `twitter`: `varchar` (Twitter handle)
- `facebook`: `varchar` (Facebook URL)
- `instagram`: `varchar` (Instagram handle)
- `created_at`: `datetime` (Company creation timestamp)
- `updated_at`: `datetime` (Company last updated timestamp)
- `industry_id`: `int` (Foreign key to industry)

#### `companies_companyreview`

- `id`: `int` (Primary Key, Auto-increment)
- `rating`: `int` (Rating of the company)
- `title`: `varchar` (Review title)
- `pros`: `longtext` (Positive aspects of the company)
- `cons`: `longtext` (Negative aspects of the company)
- `review_date`: `date` (Review submission date)
- `company_id`: `int` (Foreign key to the company)
- `user_id`: `int` (Foreign key to the user)

#### `companies_culture`

- `id`: `int` (Primary Key, Auto-increment)
- `text`: `varchar` (Description of company culture)
- `company_id`: `int` (Foreign key to company)

#### `jobs_job`

- `id`: `int` (Primary Key, Auto-increment)
- `title`: `varchar` (Job title)
- `location`: `varchar` (Job location)
- `is_remote`: `tinyint` (Whether the job is remote)
- `description`: `longtext` (Job description)
- `requirements`: `longtext` (Job requirements)
- `responsibilities`: `longtext` (Job responsibilities)
- `job_type`: `varchar` (Type of job, e.g., Full-time, Part-time)
- `experience_level`: `varchar` (Experience level required)
- `salary_min`: `decimal` (Minimum salary)
- `salary_max`: `decimal` (Maximum salary)
- `application_deadline`: `date` (Deadline for applications)
- `is_active`: `tinyint` (Is the job posting active?)
- `is_featured`: `tinyint` (Is the job featured?)
- `created_at`: `datetime` (Job posting creation timestamp)
- `updated_at`: `datetime` (Job posting last updated timestamp)
- `category_id`: `int` (Foreign key to job category)
- `company_id`: `int` (Foreign key to company)

### Summary

This project uses a relational database structure to store employer, jobseeker, company, and job-related information. The tables are connected via foreign keys to ensure data consistency and integrity. Each table has specific fields to manage user profiles, company reviews, job applications, and more.
